Descrição do Funcionamento Básico do PetShop
------------------------------------------------
O PetShop conta com alguns papéis principais:

- os pets (com dados sobre raça, nome e idade)
- os funcionários (com dados como nome, endereço e salário)
- clientes (com dados como nome, endereço e quais são os seus pets)
Além disso, existem os produtos que são utilizados no PetShop (com informações de nome do produto, valor do produto e descrição). Estes produtos são gerenciados através de um estoque, que contém uma grande quantidade de produtos armazenados.

O gerenciamento dos clientes e dos pets também devem ser realizados no sistema, pois existe um grande número destes papéis. 

Existem dois tipos de funcionários, os gestores e os veterinários, que são funcionários mas que podem possuir características e atividades definidas diferentes uns dos outros.

Finalmente, existem vários tipos de Pets que são atendidos e são atendidos e armazenados de acordo com o tipo. São eles: mamíferos, peixes e aves. Cada um destes tipos pode ter informações e ações diferentes que podem ser realizadas.

Basicamente o sistema deve permitir:

- que todos os papéis possam ser cadastrados e associados entre si. Por exemplo, deve ser possível cadastrar um novo cliente que possui 4 Pets. Naturalmente, estes Pets também deverão ser cadastrados

- que buscas sejam feitas a partir de informações armazenadas no sistema. Por exemplo, buscar todos os Pets de uma determinada raça, todos os Pets que são peixes, o endereço de determinado cliente, o salário de um funcionário etc.

- que informações sejam excluídas do sistema quando desejado

- Descrição básica das classes que serão criadas
- Descrição básica das informações dentro de cada classe, incluindo os tipos de dados a serem utilizados
- Identificação de relacionamento entre as classes definidas
- Organização das classes em diagrama UML
- Descrição básica de como realizar as possíveis ações do sistema: cadastro, busca e exclusão
